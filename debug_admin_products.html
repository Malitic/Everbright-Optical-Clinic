Admin Products Debug Guide

Follow these steps to debug the white screen issue:

Step 1: Check Browser Console

1. Open your browser's Developer Tools (F12)

2. Go to the Console tab

3. Navigate to Admin → Product Management

4. Look for any red error messages

Common errors to look for:

  • Network errors (CORS, 404, 500)
  • JavaScript errors (undefined variables, import errors)
  • React errors (component rendering issues)

Step 2: Check Network Tab

1. In Developer Tools, go to Network tab

2. Navigate to Admin → Product Management

3. Look for failed requests (red entries)

4. Check if API calls to http://127.0.0.1:8000/api/products are being made

Common Issues & Solutions

Issue 1: CORS Error

Symptom: "Access to fetch at 'http://127.0.0.1:8000' from origin 'http://localhost:8080' has been blocked by CORS policy"

Solution: CORS is already fixed, but make sure Laravel server is running

Issue 2: Network Error

Symptom: "Failed to fetch" or "ERR_CONNECTION_REFUSED"

Solution: Make sure Laravel server is running: php artisan serve

Issue 3: JavaScript Error

Symptom: "Cannot read property of undefined" or import errors

Solution: Check console for specific error and fix the component

Issue 4: Authentication Error

Symptom: 401 Unauthorized or 403 Forbidden

Solution: Make sure you're logged in as admin

Quick Fixes to Try

  1. Restart both servers:
    • Laravel: php artisan serve
    • Frontend: npm run dev
  2. Clear browser cache: Ctrl+F5 or clear localStorage
  3. Check if you're logged in: Make sure you're logged in as admin
  4. Try incognito mode: To rule out cache issues
